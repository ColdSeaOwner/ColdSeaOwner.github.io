Object Oriented Design

Posted on 2020-06-24

Objected-oriented design questions require a candidate to sketch out the classes and methods to implement technical problems or real-life objects. These problems give - or at least area believed to give an interviewer insight into your coding style.

How to create elegant, maintainable objected-oriented code.

How to Approach

  1. Handle Ambiuity

Object-oriented design (OOD) questions are often intentionally vague in order to test whether you’ll make assumptions or if you’ll ask clarifying questions. After all, a developer who just codes something without understanding what she is expected to create wastes the company’s time and money, and may create much more serious issues.

Six Ws: who, what where, when, how, why

  1. Define the Core Objects

Decide the core objects based on the information given

  1. Analyze Relationships

Analyze the relationships between the core objects. Do any objects inherit from any others? Are relationships many to many or one to many

  1. Investigate Actions

Key actions: how objects relate to each other, add some objects if necessary

Design Patterns

Capabilities and not your knowledge: Singleton and Factory Method design patterns are widely used in interviews.

It will be better to pick up a book on this topic (design patterns)

  • Singleton Class

The Singleton pattern ensures that a class has only one instance and ensures access to the instance through the application. It can be useful in cases where you have a “global” object with exactly one instance.

  • Factory Method

The Factory Method offers an interface for creating an instance of a class, with its subclasses deciding which class to instantiate.

Linked Lists

Posted on 2020-06-24

Unlike an array, a linked list does not provide constant time access to a particular “index” within the list. This means that if you’d like to find the Kth element in the list, you will need to iterate through K elements. Benefit: to add and remove items from the beginning or end of the list cost constant time

  1. Creating a Linked List
  2. Deleting a Node from a singly Linked List
  3. The “Runner” Technique: iterate through the linked list simultaneously, with one ahead of the other. The “fast” node might be ahead by a fixed amount, or it might be hopping multiple nodes for each one node that the “slow” node iterates through
  4. Recursive Problems: A number of linked list problems rely on recursion. If you are having trouble solving a linked list problem, you should explore if a recursive approach will work

Stacks and Queues

Posted on 2020-06-24

Implementing a Stack

A stack uses LIFO (last-in first-out) ordering. It uses following operations:

  • pop()
  • push(item)
  • peek()
  • isEmpty()

A stack does not offer constant-time access to the ith item. However, it does allow constant time adds and removes, as it does not require shifting elements around

One case where sacks are often useful is in certain recursive algorithms: sometimes you need to push temporary data onto stack as you recurse, and then remove them as you backtrack

Implementing a Queue

A queue implements FIFO (First-in and First-out) ordering. operations:

  • add(item)
  • remove()
  • peek()
  • isEmpty()

One place where queues are often used is in BFS or in implementing cache

937. Reorder Log Files

You have an array of logs. Each log is a space delimited string of words.

For each log, the first word in each log is an alphanumeric identifier. Then, either:

Each word after the identifier will consist only of lowercase letters, or;
Each word after the identifier will consist only of digits.
We will call these two varieties of logs letter-logs and digit-logs. It is guaranteed that each log has at least one word after its identifier.

0 <= logs.length <= 100
3 <= logs[i].length <= 100
logs[i] is guaranteed to have an identifier, and a word after the identifier.

Trees and Graphs

Posted on 2020-06-24

Type of Trees

Trees vs. Binary Trees

Binary Tree vs. Binary Search Tree

A binary search tree is a binary tree in which every node fits a specific ordering property: all left descendents <= n < all right descendents. And it cannot have duplicate values.

Balanced vs. Unbalanced

Balanced tree really means something more like “not terribly imbalanced”. It is balanced enough to ensure O(logn) times for insert and find, but it is not necessarily as balanced as it could be.

Complete Binary Trees

Filled from left to right

Full Binary Trees

Every node has either zero or two children. That is, no nodes have only one child.

Perfect Binary Trees

Both full and complete

Binary Tree Traversal

  • In-Order traversal
    • left
    • cur
    • right
  • Pre-Order traversal
    • cur
    • left
    • right
  • Post-Order traversal
    • left
    • right
    • cur

Binary Heaps (Min-Heaps and Max-Heaps)

A min heap is a complete binary tree, where each node is smaller than its children. The root is the min element in the tree.

Insert

When we insert a min-heap, we start by inserting the element at the bottom. Then we “fix” the tree by swapping the new element with its parent until we find an appropriate spot for the element.

It takes O(logn) time, where n is the number of nodes in the heap

Extract Minimum Element

Replace the top element to be the most bottom element. keep swapping it with smaller child.

Tries (Prefix Trees)

A trie is a variant of an n-ary tree in which characters are stored at each node. Each path down the tree may represent a word

A trie can check if a string is valid prefix in O(K) where K is the length of the string. Hash table also take O(K)

Graphs

A tree is actually a type of graph, but not all graphs are trees. A tree is a connected graph without cycles.

A graph is imply a collection of nodes with edges between (some of) them.

two ways to present a graph:

  • Adjacency List
  • Adjacency Matrices

Graph Search

  • DFS: use a stack
  • BFS: use a queue
  • Bidirectional Search
    • is used to find the shortest path between a source and destination node. It operates by essentially running two simultaneous BFS, one form each node. When their searches collide, we have found a path.

sudo su become root user

npm install hexo-cli -g install hexo

Start to write a blog

sudo hexo init initialize the blog (it will begin with a default theme “landscape”)

hexo s run the server at localhost:4000 (you may use this to preview blogs)

hexo n "someName" begin a new blog in source/_posts/

use IDE to edit your blogs (based on md)

Back to blog directory

hexo g generate new fiels

hexo s you may preview the blogs you write

Deploy the blog on github

Sign in to github

Use Repository name Yourgithubname.github.io

back to the terminal

npm install --save hexo-deployer-git install deployment plugin

Add a few lines bottom of config.yml

hexo d deploy, type in github password

It is now deployed

Change your theme

change config.yml theme to the name of the theme
