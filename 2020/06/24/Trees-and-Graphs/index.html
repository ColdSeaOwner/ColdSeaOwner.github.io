    Trees and Graphs

    Posted on 2020-06-24

    Type of Trees

    Trees vs. Binary Trees

    Binary Tree vs. Binary Search Tree

    A binary search tree is a binary tree in which every node fits a specific ordering property: all left descendents <= n < all right descendents. And it cannot have duplicate values.

    Balanced vs. Unbalanced

    Balanced tree really means something more like “not terribly imbalanced”. It is balanced enough to ensure O(logn) times for insert and find, but it is not necessarily as balanced as it could be.

    Complete Binary Trees

    Filled from left to right

    Full Binary Trees

    Every node has either zero or two children. That is, no nodes have only one child.

    Perfect Binary Trees

    Both full and complete

    Binary Tree Traversal

    • In-Order traversal
      • left
      • cur
      • right
    • Pre-Order traversal
      • cur
      • left
      • right
    • Post-Order traversal
      • left
      • right
      • cur

    Binary Heaps (Min-Heaps and Max-Heaps)

    A min heap is a complete binary tree, where each node is smaller than its children. The root is the min element in the tree.

    Insert

    When we insert a min-heap, we start by inserting the element at the bottom. Then we “fix” the tree by swapping the new element with its parent until we find an appropriate spot for the element.

    It takes O(logn) time, where n is the number of nodes in the heap

    Extract Minimum Element

    Replace the top element to be the most bottom element. keep swapping it with smaller child.

    Tries (Prefix Trees)

    A trie is a variant of an n-ary tree in which characters are stored at each node. Each path down the tree may represent a word

    A trie can check if a string is valid prefix in O(K) where K is the length of the string. Hash table also take O(K)

    Graphs

    A tree is actually a type of graph, but not all graphs are trees. A tree is a connected graph without cycles.

    A graph is imply a collection of nodes with edges between (some of) them.

    two ways to present a graph:

    • Adjacency List
    • Adjacency Matrices

    Graph Search

    • DFS: use a stack
    • BFS: use a queue
    • Bidirectional Search
      • is used to find the shortest path between a source and destination node. It operates by essentially running two simultaneous BFS, one form each node. When their searches collide, we have found a path.
